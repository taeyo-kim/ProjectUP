[In development]Vaulted Backups for Azure Disk
  • Description:Today, Azure Disk Backup protects data by taking regular, crash-consistent snapshots of Azure Disks. These snapshots are stored within your subscription and tenant in a resource group, referred to as the Operational Tier in Azure Backup. This enables fast, operational recovery for scenarios such as accidental deletion or data corruption and is commonly used alongside Azure VM Backup, which provides application-aware protection for virtual machines.  
    Backup best practices recommend a 3-2-1 strategy, including storing a copy of backups offsite in an isolated location securing against a tenant-level compromise. 

    With the introduction of Vault Tier backups, Azure Disk Backup now complements Azure VM Backup by extending disk-level protection with vault-based isolation, independent access controls, and immutability. This enables recovery of trusted data during ransomware attacks or tenant compromises, bringing disk backups in line with Azure VM Backup grade cyber-recovery security.  

    As part of this Private Preview, Azure Disk Backup introduces two key capabilities: 

    • Vault Tier Backup: Protect disk backups in a vault (offsite) to meet cyber-recovery, ransomware resilience, and compliance requirements. 

    • Regional Disaster Recovery: Restore disk backups to a paired Azure region, enabling new disaster-recovery scenarios alongside Azure VM Backup and Azure Site Recovery.

[In preview]Azure Monitor pipeline data transformations
  • Description:Azure Monitor pipeline data transformations feature is now Public Preview! This feature enables users to shape telemetry before it’s ingested into Azure Monitor, helping them control ingestion costs, improve data quality, and simplify analytics at scale.
    Built into the Azure Monitor pipeline for edge and multi-cloud environments, transformations allow data types such as Syslog and CEF to be filtered, aggregated, standardized, and reshaped prior to ingestion. Automatic schema standardization and built-in validation guardrails ensure continued compatibility with standard tables, allowing users to apply transformations confidently without disrupting dataflow.
    Transformations also include prebuilt KQL templates for common scenarios, along with advanced filtering and aggregation to reduce noise, lower ingestion volume, and summarize high frequency data into meaningful time windows.By shaping data closer to the source, Azure Monitor pipeline data transformations deliver cleaner data and faster insights for high volume and complex telemetry environments.
